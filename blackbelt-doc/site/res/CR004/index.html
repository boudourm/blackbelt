CR004 Démarrage de BlackBelt avec Thomas CAIATI

Date : 06-07-2020
Type : Réunion
Lieu : Pitet, SAMSE Groupe Headquarters, Echirolles
Organisateur : Thomas CAIATI (Ingénieur Informatique à la SAMSE)
Rapporteur : Mehdi BOUDOUR (Étudiant en Master MIAGE à IM2AG)
Présents : Thomas CAIATI, Mehdi BOUDOUR

Attention :
Les spécifications énoncées sont admisent jusqu'à redéfinition dans les prochains comptes rendus.
  1. Première solution envisageable suite à l’études de l’implementation SpringFox de Swagger est d’exploiter la représentation Json du API Controler émise par SpringFox.
  2. Le Json récupéré sera donc traduit en test Karaté.
  3. Les tests seront générés pour chaque endpoint d’un API individuellement en fichiers Karaté.
  4. Un endpoint sera créé pour chaque génération de tests.
  5. Une fois les tests individuellement créés, un endpoint supplémentaire doit être mit en place afin de générer l’ensemble des tests de tous les endpoint d’un API
  6. La transformation doit se faire en Java pure avec pour entrée le Json SpringFox et sortie les fichier Karaté de test.
  7. Une fois le parse Json To Karaté terminé on envisage la création d’un custom plugIn SpringFox afin de documenter les endpoint de tests sur la SpringFox UI.
  8. Les fichiers de tests Karaté sont structurés en feature.
  9. Chaque feature contient un ou plusieurs scénarios de tests.
  10. Chaque Scénario comporte différentes lignes de tests unitaires écrites en langage cucumber.
